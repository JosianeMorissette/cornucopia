Generate type checked Rust

from your PostgreSQL

Cornucopia is a small CLI utility resting on tokio-postgres designed to facilitate PostgreSQL workflows in Rust.

Try it yourself!

SQL-first

Your database schema is the source of truth. No ORM.

Custom user types

Support custom user types like composites, enums, domains and one-dimensional array

Asynchronous streams

Strongly-typed async row streams suitable for web servers.

Nullable return columns

Allow the user to have nullable return colums.

Self managed

Build your queries against your own live database, or let Cornucopia manage that for you.

Flexible connexion types

Use, mix and match the connection types that you want: pooled or not, transaction or not.

Compatible with build.rs

Rebuild Rust queries whenever SQL files change.

No macros

No macros, respects your compile times.

Cornucopia is designed to
facilitate PostgreSQL in Rust

  • Cornucopia aims to get out of your way, transpiling your PostgreSQL queries to Rust on demand.
  • Each query is prepared against your schema, ensuring that the query statements are valid SQL.
  • These prepared statements are then be used to generate properly type-checked Rust code for this query.
